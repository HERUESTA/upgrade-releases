# 今週のRails
- https://rubyonrails.org/2026/3/23/Rails-Versions-7-2-3-1-8-0-4-1-and-8-1-2-1-have-been-released
## Railsのバージョン7.2.3.1, 8.0.4.1, 8.1.2.1がリリースされました🎉
### 脆弱性対応でパッチバージョンがリリースされた
- 開発環境で、例外がエスケープ処理されずに出力されてしまう不具合の改修
  - ⚠️ローカルサーバーでサーバーを動かしている開発者が悪意のあるURLをクリックした場合に、影響を受ける可能性あり
### ActionPack, Action View, およびActive Supportにおける複数のXSS（クロスサイトスクリプティング）脆弱性の可能性
- Action Viewタグヘルパーで空の文字列を使用すると、不正なHTMLが生成されてしまう
  - ActionPack：Webリクエスト処理とレスポンスの生成を担当する部分
  - Action View：Controllerから受け取ったレスポンスをレンダリングする
    - レンダリング：もとになる情報を整形して表示する
  - Active Support：Rubyの標準クラスを便利に拡張するメソッド群を提供するもの
- Active Storageにおけるレンジリクエストに関連した2件の潜在的なDos脆弱性
  - Active Storage：Amazon S3やGoogle Cloudなどのクラウドストレージサービスへのファイルのアップロードや、ファイルをActive Recordオブジェクトにアタッチする機能
    - Active Record：MVCのMの部分で、データとビジネスロジックを表現するシステムの階層
    - アタッチ：モデルインスタンスに紐づける
      - ex):`User`モデルにアバター画像を紐づける場合、モデル側で`has_one_attached :avatar`と宣言すると、`user.avatar`でいつでもアクセスができるようになる
### Active Supportにおける数値のフォーマットに関連した、2件の潜在的なDos脆弱性
- `NumberToDelimitedCounter`が`gsub!`の正規表現を行なっていたが、非常に長い数字の文字列に関して、計算量が多くなってしまう
### Active Storage DiskServiceにおけるパス・トラバーサルおよびグロブ・インジェクションの脆弱性の可能性
  - Active Storage DiskService：アップロードしたファイルをクラウドではなくローカルに保存するサービス
  - glob injection：ファイルパスのパターンマッチに使われる「glob」の仕組みを悪用する攻撃方法
  - Path traversal：`../`などを利用して、本来許可されていないディレクトリやファイルに不正にアクセスする攻撃手法
### Active Storageの直接アップロードにおけるメタデータのフィルタリング不備
- HTTP Rangeヘッダーにおけるバイト範囲の数を制限しておらず、通常よりも多くのCPU使用率になりDos攻撃につながる可能性がある
